## Active Integrations

### 1. GitHub Pages (Hosting)

**Purpose:** Free static site hosting and automatic deployment

**How It Works:**
1. Push to `main` branch
2. GitHub Pages automatically detects Jekyll project
3. Runs build via GitHub Actions
4. Deploys static output to `https://username.github.io/learn-with-satya/`

**Configuration:**
- Repo must be public (for free tier)
- Setting: Repo → Settings → Pages → Source: "GitHub Actions" or "Deploy from branch"
- Deploy time: 1–3 minutes

**Limitations:**
- Jekyll plugins limited to [GitHub Pages whitelist](https://pages.github.com/versions/)
- Custom Ruby plugins not supported
- No server-side processing

**Documentation:** https://docs.github.com/en/pages

---

### 2. GitHub Actions (CI/CD)

**Purpose:** Automated build and deployment on each push

**Workflow File:** `.github/workflows/acquire-codebase-knowledge.md` 
(Note: Currently only workflow file found is this acquisition guide; full deploy workflow may be in `.github/workflows/` if created)

**Typical Pipeline:**
```yaml
name: Deploy Jekyll to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.1'
          bundler-cache: true
      - uses: actions/jekyll-build-pages@v1
      - uses: actions/upload-pages-artifact@v1
      - uses: actions/deploy-pages@v1
```

**Triggers:**
- on: push to `main` branch
- (Optional) on: schedule (e.g., nightly rebuild)

**No Configuration Required:** GitHub Pages automatically runs Jekyll if a Gemfile/\_config.yml exists

### 3. Lunr.js (Client-Side Search)

**Purpose:** Full-text search index built at Jekyll build time

**How It Works:**
1. Jekyll generates `search.json` at build time containing all post metadata
2. `assets/js/search.js` loads this index on page load
3. User types in search box → Lunr queries index in browser
4. Results displayed instantly (< 100ms)

**Configuration:**
- Search index generated by Jekyll plugin(s)
- Index location: `/search.json`
- JavaScript: `assets/js/search.js` (manages UI and queries)

**Data Included in Index:**
- Post title
- Post excerpt
- Post URL
- Post category
- Post tags
- Series information (if applicable)

**Limitations:**
- Index size grows with post count (typically < 200KB for 500 posts)
- No real-time indexing (rebuilds only on deploy)
- No fuzzy search across variants (but supports stemming)

**Documentation:** https://lunrjs.com/

### 4. BrowserSync (Local Development)

**Purpose:** Local dev server with live reload

**How It Works:**
1. Run `npm run dev`
2. Gulp starts BrowserSync
3. Site served at `http://localhost:3000`
4. File changes trigger rebuild + browser refresh automatically

**Features:**
- Live reload on any file change (Markdown, YAML, CSS, JS)
- Multiple device testing (shows same page on multiple screens synced)
- Network error handling

**Configuration:** `gulpfile.js`
```javascript
browserSync.init({
  server: './_site/',
  port: 3000,
  open: false
});
```

**Documentation:** https://browsersync.io/

## Planned Integrations

### 1. Claude API [PLANNED]

**Purpose:** AI-powered autonomous blog post generation

**Phase:** Phase 3 of project roadmap (Agent Pipeline)

**Model:** `claude-sonnet-4-6`

**Usage:**
```javascript
const Anthropic = require("@anthropic-ai/sdk");

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

const message = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [
    { role: "user", content: "Research this topic..." },
  ],
});
```

**Integration Points:**
1. **Orchestrator Agent** - Classify topic, select category/series
2. **Research Agent** - Uses Claude's `web_search` tool to fetch references
3. **SEO Agent** - Design outline and keywords
4. **Writer Agent** - Compose full post body
5. **Formatter Agent** - Assemble Jekyll-ready `.md` file

**Authentication:**
```env
# .env file (NOT COMMITTED)
ANTHROPIC_API_KEY=sk-ant-v0-xxxxxxxx
```

**Limitations:**
- Rate limits: Plan tier-dependent (typically 100K tokens/day for free tier)
- Cost: $0.003/1K input tokens, $0.015/1K output tokens
- Latency: 2–5 seconds per request (acceptable for async content gen)

**Documentation:** https://docs.anthropic.com/

### 2. Unsplash API [PLANNED]

**Purpose:** Find and attribute free images for blog posts

**Phase:** Phase 3–4 (Writer/Formatter agents)

**Usage Pattern:**
```javascript
const fetch = require("node-fetch");

// Writer Agent: Get image recommendations
const response = await fetch(
  `https://api.unsplash.com/search/photos?query=transformer+diagram&client_id=${process.env.UNSPLASH_ACCESS_KEY}`
);
const results = await response.json();

// Results include: url, photographer name, photo page, license info
```

**Integration Points:**
1. **SEO Agent** - Generates image search queries
2. **Writer Agent** - Fetches suggestions, embeds URLs
3. **Formatter Agent** - Validates attribution block

**Data Returned:**
```json
{
  "results": [
    {
      "urls": {
        "raw": "https://images.unsplash.com/photo-xyz",
        "full": "https://images.unsplash.com/photo-xyz?...",
        "regular": "https://images.unsplash.com/photo-xyz?w=1080&..."
      },
      "user": {
        "name": "Jane Doe",
        "portfolio_url": "https://jdoe.com"
      },
      "links": {
        "html": "https://unsplash.com/photos/abc123"
      }
    }
  ]
}
```

**Attribution Format:**
```markdown
![Alt text](https://images.unsplash.com/photo-xyz)
*Photo by Jane Doe on [Unsplash](https://unsplash.com/photos/abc123) — Unsplash License*
```

**Authentication:**
```env
UNSPLASH_ACCESS_KEY=your-access-key
```

**Free Tier Limits:**
- 50 requests/hour (per IP)
- Unlimited searches
- No cost

**Documentation:** https://unsplash.com/api

**Hero Image Handling:**
- Unsplash provides suggestions
- Author manually sources and places hero image
- System provides formatted Markdown with attribution

### 3. Giscus (GitHub Discussions Comments) [PLANNED]

**Purpose:** Comment system powered by GitHub Discussions

**Phase:** Phase 5 (Polish & SEO)

**How It Works:**
1. User clicks "Comment" on a post
2. GitHub OAuth login (if not already logged in)
3. Comment appears in post + in GitHub Discussions thread
4. Two-way sync: edits in GitHub show in post

**Setup:**
1. Enable Discussions in repo settings
2. Install [Giscus app](https://github.com/apps/giscus) on repo
3. Get configuration code:
   ```html
   <script src="https://giscus.app/client.js"
           data-repo="yourusername/learn-with-satya"
           data-repo-id="R_xxx"
           data-category="Comments"
           data-category-id="DIC_xxx"
           data-mapping="pathname"
           data-reactions-enabled="1"
           data-emit-metadata="0"
           data-input-position="bottom"
           data-theme="preferred_color_scheme"
           data-lang="en"
           crossorigin="anonymous"
           async>
   </script>
   ```
4. Add `giscus` include to `_layouts/post.html`

**Advantages:**
- No custom backend required
- Leverages existing GitHub accounts
- Moderation via GitHub Issues
- Comments stored in Discussions tab

**Limitations:**
- Requires user to have GitHub account
- Requires public repo
- Discussions must be enabled

**Documentation:** https://giscus.app/

### 4. Plausible Analytics [PLANNED]

**Purpose:** Privacy-first visitor analytics

**Phase:** Phase 5 (Polish & SEO)

**How It Works:**
1. Add simple script tag to site header
2. Tracks pageviews, referrers, countries (without cookies)
3. GDPR-compliant (no consent banner needed)
4. Dashboard at plausible.io

**Setup:**
```html
<!-- In _includes/head.html -->
<script defer data-domain="yourusername.github.io" src="https://plausible.io/js/script.js"></script>
```

**Features Tracked:**
- Pageviews
- Bounce rate
- Average session duration
- Traffic source (referrer)
- Device type
- Geographic location
- Top pages
- Search queries (if applicable)

**Cost:**
- Free tier: Limited data
- Paid (starting $9/mo): Full features, custom domain

**Advantages Over Google Analytics:**
- Privacy-first (GDPR, CCPA compliant)
- No cookie banner required
- Faster page load (lighter script)
- Simple dashboard
- Email reports

**Documentation:** https://plausible.io/docs

## Security & Credentials

### Credential Management

**API Keys (Never Committed):**
- Store in `.env` file (listed in `.gitignore`)
- Load via `process.env.KEY_NAME` in agent scripts
- Rotate keys annually or immediately if leaked

**GitHub Actions Secrets:**
- If agents run in GitHub Actions: use Secrets
- Syntax: `${{ secrets.ANTHROPIC_API_KEY }}`
- Set in Repo → Settings → Secrets

**Example `.env` (NOT IN GIT):**
```env
ANTHROPIC_API_KEY=sk-ant-v0-xxxxxxxx
UNSPLASH_ACCESS_KEY=xxxxxxxx
```

**Example `.gitignore` Entry:**
```
.env
.env.local
*.secret
```

## Monitoring & Troubleshooting

### GitHub Actions Build Failures

**Check Logs:**
1. Go to Repo → Actions
2. Click failed workflow run
3. Expand "Build" step
4. Look for Jekyll or Gulp errors

**Common Issues:**
- **Gemfile.lock mismatch:** Delete lock file, re-run
- **YAML parse error:** Validate front-matter syntax
- **Missing gem:** Check Gemfile dependencies

### Search Index Issues

**Lunr.js not working:**
1. Check browser console for JavaScript errors
2. Verify `/search.json` exists in `_site/`
3. Ensure search.js loads successfully (network tab)

### API Rate Limits (When Active)

**Claude API:**
- Monitor token usage in Anthropic dashboard
- Implement backoff/retry logic
- Consider request throttling if frequent

**Unsplash API:**
- 50 requests/hour per IP
- Implement caching to avoid repeated requests
